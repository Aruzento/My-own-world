# UX / Onboarding Checklist

Этот документ закрывает пункт 16 плана и задает минимальный стандарт для будущего onboarding.

## Цель

Пользователь должен понять продукт без чтения истории разработки:

- где выбрать workspace;
- чем карточка отличается от карты и task tracker;
- как работает дерево;
- как связаны wiki-links;
- что проверять после изменений UI.

## Sample Workspace

Пример workspace лежит в `docs/sample-workspace`.

Его можно открыть как обычную папку workspace. Внутри есть:

- `pages/0001-welcome.md` - стартовая карточка;
- `pages/0002-campaign-map.md` - пример карты;
- `pages/0003-task-tracker.md` - пример task tracker;
- `assets/.gitkeep` - пустая папка для будущих изображений.

## Стартовый Tutorial

Стартовый tutorial доступен внутри приложения:

1. Нажать кнопку `Инструменты` в верхней тонкой панели.
2. Выбрать `Быстрый старт`.
3. Пройти четыре шага: workspace, первая сущность, дерево, wiki-links.

## Как Устроен Продукт

В том же popup `Инструменты` есть раздел `Как устроено`.

Он объясняет:

- карточки;
- карту кампании;
- task tracker;
- runtime UI.

## Onboarding По Системам

Минимальные темы, которые должны оставаться понятными пользователю:

- карточки: тип, теги, alias, блоки и свойства;
- дерево: parent, order, переносы, подсветка текущей сущности;
- wiki-links: `[[Название]]`, alias lookup, сохранение видимого текста;
- карта: фон, grid, fog, tokens, shapes, presentation;
- task tracker: колонки, задачи, drag and drop.

## UX Checklist Перед Изменением

- Popup не выходит за видимую область экрана.
- Текст не обрезается и не налезает на иконки.
- Кнопки имеют понятный hover/focus.
- Основное действие доступно без знания горячих клавиш.
- Runtime UI не попадает в сохраненный HTML.
- После изменения карты проверены save/reload и presentation sync.
- После изменения карточек проверены wiki-links, undo/redo, plain-text paste и clean-save.
- После изменения task tracker проверены задачи, колонки, drag and drop и reload.

## Definition Of Done Для Пункта 16

- Есть sample workspace.
- Есть встроенный быстрый старт.
- Есть встроенное описание основных частей продукта.
- Есть checklist для карточек, дерева, wiki-links, карты и task tracker.
- `README.md`, `PLANS_AND_TECH_DEBT.md`, `WORK_LOG.md` и manual обновлены.
