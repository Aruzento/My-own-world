# Брендбук MyOwnWorld

Дата обновления: 15.06.2026

## Идея Бренда

MyOwnWorld - это рабочий кабинет мастера НРИ.

Интерфейс должен ощущаться как место, где хранятся карты, заметки, существа, правила и решения кампании. Он не должен выглядеть как яркая игровая менюшка. Главная эмоция: спокойная власть над большим живым миром.

Ключевые ощущения:

- теплый темный кабинет;
- пергамент и латунь, но без декоративной перегрузки;
- древо знаний, а не набор окон;
- инструмент для долгой работы;
- мягкая магия, которая помогает, но не отвлекает.

## Палитра

Основная палитра называется **Archive Hearth**.

Цвета:

- `Deep Umber` - основной фон: `#12100d`.
- `Charred Walnut` - панели и elevated surfaces: `rgba(35, 31, 25, 0.78)`.
- `Warm Parchment` - основной текст: `rgba(255, 248, 232, 0.96)`.
- `Muted Ink` - вторичный текст: `rgba(226, 209, 178, 0.70)`.
- `Candle Gold` - основной акцент: `#f4ca76`.
- `Moss Seal` - вторичный активный/выбранный акцент: `#8fae84`.
- `Ruby Wax` - опасные действия: `#e88980`.

Правила:

1. Синий не использовать как системный акцент, focus, checkbox или hover.
2. Желтый акцент используется для наведения, фокуса, добавления, перемещения, выбора будущей позиции.
3. Зеленовато-мшистый акцент используется для выбранного состояния, активного режима и подтвержденной связи.
4. Красный используется только для закрыть, удалить, опасно, ошибка.
5. Цвет никогда не должен быть единственным объяснением действия: у действия должен быть текст, иконка или состояние.

## Поверхности

У приложения четыре уровня поверхности:

1. `App background` - темный теплый фон приложения.
2. `Panel` - sidebar, editor, основные рабочие зоны.
3. `Elevated` - popup, floating toolbar, меню.
4. `Embedded` - блоки, поля, карточки внутри редактора.

Правила:

- панели не должны быть яркими;
- popup должен быть мягко отделен тенью и границей;
- блок не должен превращаться в отдельную “карточку внутри карточки”, если он не repeated item;
- рабочие поверхности должны быть плотными, но с понятным воздухом.

## Типографика

Тон текста:

- короткие названия;
- без маркетинговых описаний внутри рабочего UI;
- пояснения только там, где пользователь реально принимает решение.

Заголовки:

- крупный заголовок только для карточки, карты или отдельной сущности;
- внутри popup и блоков использовать компактные заголовки;
- не использовать отрицательный letter-spacing как декоративный прием.

## Motion

Анимации должны помогать руке понимать интерфейс.

Разрешено:

- микронажатие кнопок;
- мягкое появление popup;
- плавное раскрытие дерева/панелей;
- placeholder при drag and drop;
- подсветка выбранного элемента;
- плавный hover.

Запрещено:

- долгие анимации;
- анимации, которые задерживают ввод;
- heavy blur на карте, токенах, fog и больших слоях;
- анимации drag/pan/zoom карты, ухудшающие производительность;
- motion без уважения `prefers-reduced-motion`.

## Компоненты

### Блоки

Блок должен:

- иметь компактный заголовок;
- иметь runtime controls, которые появляются мягко и не сохраняются;
- использовать теплую границу и легкий hover;
- не ломать текстовое редактирование.

### Popup

Popup должен:

- помещаться в viewport;
- закрываться повторным нажатием на кнопку, Escape и outside click, если сценарий это позволяет;
- быть перетаскиваемым за свободное место;
- использовать `ui-panel` или визуально совместимый стиль.

### Вкладки И Панели

Будущие вкладки должны:

- иметь активное состояние через `Moss Seal`;
- показывать hover через `Candle Gold`;
- не использовать синий underline;
- быть плотными, чтобы не красть место у рабочей области.

### Карта

Карта - производительная зона.

Для карты разрешено:

- подсветка выбранных токенов;
- компактные toolbar states;
- короткие popup;
- вектор движения;
- мягкие hover для controls.

Для карты запрещено:

- анимировать весь background;
- анимировать fog при рисовании;
- добавлять тяжелые box-shadow на большое количество токенов;
- делать motion, который мешает перетаскиванию.

## Файлы Бренда

- `styles/variables.css` - базовые CSS-переменные и фон.
- `styles/design-tokens.css` - системные токены.
- `styles/brand-system.css` - финальная кожа, motion и общие состояния.
- `docs/02-architecture/ui/DESIGN_SYSTEM_CONTRACT.md` - инженерный контракт.

## Проверка Будущего UI

Перед завершением задачи спросить:

1. UI выглядит как рабочий кабинет мастера, а не как случайный web-app?
2. Нет ли синего системного акцента?
3. Есть ли hover/focus/active/disabled?
4. Popup помещается в экран?
5. Drag/drop показывает будущее место без дрожания?
6. Карта не получила тяжелых анимаций?
7. Текст не обрезается там, где он нужен для выбора?
