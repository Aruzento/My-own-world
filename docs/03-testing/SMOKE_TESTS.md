# Smoke And Regression Tests

Этот чеклист нужен перед коммитом после изменений в editor, storage, tree, campaign map или block system.

## Автотесты

1. Выполнить `npm run verify`.
2. Если нужно проверить только синтаксис JS, выполнить `npm run check:js`.
3. Если нужно запустить только unit-тесты, выполнить `npm test`.
4. Если нужно запустить браузерный smoke, выполнить `npm run test:browser`.
5. Убедиться, что тесты не требуют выбранного workspace и не меняют пользовательские `.md` файлы.

Для точечного browser smoke можно передать аргументы Playwright после `--`:

```powershell
npm run test:browser -- --grep schema-recovery
npm run test:browser -- tests/browser/campaign-map-initiative.spec.mjs
```

Первые автотесты проверяют model/helper-слои без браузера:

- нормализацию и действия Task Tracker;
- нормализацию Campaign Map model;
- правила уникальных названий страниц.

`npm run verify` выполняет:

- проверку синтаксиса всех `.js` и `.mjs` файлов;
- unit-тесты;
- `git diff --check`;
- проверку `docs/MY_OWN_WORLD_FULL_MANUAL.docx` как zip-архива.

Браузерные сценарии постепенно переводятся из ручного smoke-чеклиста в Playwright.

Browser smoke хранится в `tests/browser/`:

- `tests/browser/README.md` описывает правила browser runner;
- `tests/browser/scenarios.mjs` хранит приоритетный список сценариев, которые нужно автоматизировать первыми.
- `tests/browser/app-shell.spec.mjs` проверяет старт приложения без workspace и console/page errors.

Первые browser-сценарии для автоматизации:

1. `tree-dnd-save-after-move`;
2. `tree-collapse-persistence`;
3. `campaign-map-token-flow`;
4. `campaign-map-presentation-sync`;
5. `toolbar-formatting-boundary`;
6. `task-tracker-dnd-persistence`;
7. `popup-viewport-fit`;
8. `template-create-card`.

## Базовый Запуск

1. Запустить static server из папки приложения.
2. Открыть `http://127.0.0.1:5173/`.
3. Проверить, что консоль браузера не содержит ошибок модулей.
4. Выбрать workspace вне папки приложения.

## Карточки

1. Создать карточку.
2. Убедиться, что новая карточка сразу открылась.
3. Сменить тип карточки.
4. Добавить tag и alias.
5. Добавить wiki-link через `[[...]]`.
6. Обновить страницу браузера.
7. Убедиться, что title, type, tags, aliases и wiki-link сохранились.

## Блоки

1. Добавить текстовый блок.
2. Добавить item block.
3. Добавить spells block.
4. Добавить image block, загрузить картинку, проверить удаление и кадрирование.
5. Добавить `Стат. блок DnD`.
6. В DnD block изменить характеристику и проверить авторасчёт навыков.
7. Вручную изменить навык, убедиться, что поле стало ручным.
8. Очистить ручной навык, убедиться, что авторасчёт вернулся.
9. Сохранить, обновить страницу, открыть карточку снова.

## Таблицы

1. Добавить таблицу.
2. Добавить строку.
3. Вставить несколько строк текста через paste.
4. Проверить Enter и Shift+Enter.
5. Сохранить и открыть заново.

## Дерево

1. Создать несколько родительских карточек с дочерними.
2. Свернуть родительские ветки.
3. Обновить страницу через `Ctrl + F5`.
4. Проверить, что ветки остались свернутыми.
5. Pointer-drag карточку внутрь другой ветки.
6. Pointer-drag карточку на тот же уровень для сортировки.
7. Pointer-drag карточку на placeholder `перенести сюда`.
8. Открыть карточку, перенести ее в дереве, изменить текст без переоткрытия и обновить страницу.
9. Убедиться, что изменение сохранилось в новом месте.
10. Нажать `Дерево` в левом rail, убедиться что дерево скрывается, editor расширяется, затем нажать `Дерево` еще раз.
11. Удалить дочернюю карточку и обновить страницу.

## Popup И UI

1. Открыть меню создания карточки кнопкой `+` в строке `Корень`.
2. Нажать `+` повторно и убедиться, что меню закрылось.
3. Открыть block popup через добавление блока и закрыть его кликом вне popup.
4. Открыть wiki create menu для отсутствующей wiki-link и закрыть через `Escape`.
5. Навести курсор на существующую wiki-link и проверить preview.
6. Открыть profile popup и закрыть повторным кликом по профилю.

## Карта Кампании

1. Создать карту.
2. Сменить изображение карты.
3. Включить сетку и изменить размер сетки.
4. Добавить существо из карточки.
5. Добавить объект из карточки.
6. Переместить существо и объект.
7. Проверить, что вектор перемещения появляется только при расстоянии больше `0 ft`.
8. Открыть презентацию и убедиться, что позиции совпадают.
9. Нарисовать туман, стереть часть, нажать Fog all и Unfog all.
10. Добавить квадрат, треугольник и круг.
11. Переместить фигуры и изменить размеры/вершины.
12. Обновить страницу и проверить карту снова.
13. Перетащить карточку существа из дерева на карту.
14. Проверить, что появилась сущность на карте и дочерний дубль в дереве.
15. Скрыть сущность, подвигать ее и проверить, что презентация не показывает скрытый вектор.

## Task Tracker

1. Создать Task Tracker.
2. Создать задачу через `+` внутри колонки.
3. Изменить название, описание и checklist.
4. Перетащить задачу между колонками.
5. Перетащить колонку.
6. Удалить задачу.
7. Создать новую задачу через общий create menu и выбрать tracker.
8. Обновить страницу и проверить, что данные сохранились.

## Шаблоны

1. Создать карточку с типом, тегами, alias, описанием и блоком.
2. Через меню дерева сделать карточку шаблоном.
3. Создать карточку по шаблону.
4. Проверить, что новая карточка появилась на уровне текущей открытой карточки.
5. Удалить шаблон из popup-а выбора шаблона.

## Сохранение

1. Быстро выполнить несколько действий подряд: drag token, изменить HP, изменить fog.
2. Обновить страницу.
3. Проверить, что последнее состояние не потерялось.

## UTF-8

Перед коммитом выполнить проверку diff на типичные mojibake-маркеры из правила проекта. В документацию не добавлять сами маркеры как обычный текст, чтобы проверка не давала ложные срабатывания.
