# Архив экспериментов

Этот файл хранит идеи, которые уже пробовали, но временно вывели из активного интерфейса.
Важно: архив не означает "удалить навсегда". Это означает "идея требует переосмысления перед продолжением".

## Стат. блок DnD v 2.0

### Что пробовали

- Новый блок `dndStatsV2` как более полный лист персонажа.
- Поля происхождения: раса, класс, вид, подкласс.
- Боевые показатели: КЗ, хиты, скорость, уровень, опыт, кости хитов, хиты от смерти.
- Характеристики, навыки и спасброски с авторасчётом.
- Тёмное оформление под общий стиль приложения.

### Почему архивировано

- Блок начал превращаться в отдельную большую систему правил.
- Было слишком много ручных полей, но ещё не было нормальной модели данных для рас, классов, бонусов и формул.
- Визуально и архитектурно это стало похоже на недодуманную промежуточную версию, а не на устойчивую основу.

### Что оставить на будущее

- Идея полного листа персонажа нужна.
- Идея уже перенесена в текущую архитектуру: полный лист строится как runtime-витрина `CharacterModel`, а игровые значения берутся из `Свойств` карточки.
- Дальнейшее развитие должно идти через редактируемый `CharacterModel` / `CardVariablesModel`, а не через один гигантский HTML-блок.

## Блок "Переменные"

### Что пробовали

- Новый блок `variables`.
- Каталог системных переменных.
- Саб-блоки переменных: происхождение, характеристики, расчётные характеристики, боевые показатели, хиты.
- Select по карточкам с тегом `race`.
- MVP расчёта: базовая характеристика персонажа + характеристика выбранной расы.

### Почему архивировано

- Идея правильная, но первая версия выглядела как черновик formula engine.
- Не хватало отдельной модели данных `VariableModel`.
- Не было языка формул вроде `base.str + race.str + class.str`.
- Не было понятного UI для зависимости переменных от других карточек.

### Что оставить на будущее

- Идея переменных встроена в `PropertiesModel` и `CardVariablesModel`: свойства карточки являются переменными сущности.
- Отдельно позже нужно вернуться не к старому UI-блоку, а к следующему уровню:
  - расчетные переменные;
  - формулы;
  - зависимости между карточками;
  - Rule Tree provider;
  - UI просмотра зависимостей.

## Текущий статус

- Оба блока убраны из активного выбора блоков.
- Их runtime setup/render не подключаются в `app.js` и `editor.js`.
- Их CSS не импортируется в `styles/main.css`.
- Исходники сохранены в проекте как архивная база для будущего переосмысления.
- Практическая линия развития перенесена в `PropertiesModel`, `CardVariablesModel` и `CharacterModel`.
